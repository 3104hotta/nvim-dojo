# 最終総合演習 — 設計メモ（未決定事項）

中級編・上級編に共通する設計上の課題をまとめる。

---

## 背景

中級編・上級編をそれぞれ 3問（15〜30分 or 30〜60分） から
**12問 × 8〜15分** に再構成した（3テーマ × 4小問の構造）。

題材ファイルは元の3つ（`exercise-1/`・`exercise-2/`・`exercise-3/`）を流用しており、
小問だけを README で分割している。

---

## 課題：小問間の依存関係

中級・上級ともに、テーマC は完全独立だが、テーマA・B は前の小問の完了状態を前提とする。

### 中級編

| テーマ | 独立性 | 理由 |
|--------|--------|------|
| A（log → tracing 移行）| ⚠️ 前から順 | 全5ファイルを段階的に変換するため、A2 は A1 の結果を前提 |
| B（3ペイン同時編集）| ⚠️ 前から順 | `model.rs` の変更を `db.rs`・`api.rs` が引き継ぐ |
| C（マクロで10ファイル変換）| ✅ 完全独立 | C1 は1ファイルだけ、C2 以降も独立して再実行可能 |

### 上級編

| テーマ | 独立性 | 理由 |
|--------|--------|------|
| A（OSS リファクタ）| ⚠️ 前から順 | 全部 `exercise-1/src/main.rs` 1ファイルを段階的に変更 |
| B（v1 → v2 移行）| ⚠️ 前から順 | `v1/src/lib.rs` を段階的に v2 化 |
| C（HTTP サーバー実装）| ✅ 完全独立 | 各小問が別ファイル（config.rs / handler.rs / router.rs / main.rs）|

要件定義の「各ミッションの題材は独立していて、前のミッションの結果に依存しない」と
テーマA・Bがずれている。

---

## 検討した3案

| 案 | 工数 | 良い点 | 悪い点 |
|----|------|-------|-------|
| ① そのまま + `reset.sh` 追加 | 小 | ファイル増えない。タイムアタック再挑戦が簡単 | A2 だけやるには A1 を毎回やる必要あり |
| ② 各小問に開始状態ファイルを用意（`A2-start.rs` など）| 中 | 真の独立性 | 似たファイルが増える（A1〜A4 で4状態 × 中級＋上級 = 計16状態程度） |
| ③ 題材を作り直して各小問を別ファイルに | 大 | 完全に独立 | テーマの一貫性が薄れる |

---

## 採用方針：①（決定）

2026-04-26 に確定。

理由:
- タイムアタック再挑戦時に元の状態へ即戻せる
- テーマA・Bは「順を追って段階的にリファクタする流れ」自体が学びの一部
- 「今日はA1だけ」「明日A2から」も `git stash` / コミット運用で対応可能

実装内容:
- `final-exam/intermediate/reset.sh` 追加（`git checkout HEAD -- exercise-*` ベース）
- `final-exam/advanced/reset.sh` 追加
- 中級・上級それぞれの README で、テーマA・B の冒頭に「前の小問の完了状態を前提」と明記し、
  再挑戦時は `reset.sh` を案内

注意: `reset.sh` は **題材ファイルが git にコミット済み** であることが前提。
リポジトリ初期化直後はまず `git add final-exam && git commit` してから使う。
