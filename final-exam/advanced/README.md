## 問題 1 — OSS コードの大規模リファクタ

`exercise-1/` は実際の OSS プロジェクトから抜粋したコード（約500行）。
以下のリファクタをすべて nvim 単体で完了させよ。

### 条件

- LSP（rust-analyzer）をフル活用する
- プラグインなし・マウスなし

### 変更内容

1. `Error` 型を `thiserror` ベースから独自実装の `AppError` enum に移行する（`rename` + `code action`）
2. `async fn` を使っている関数の `await` 呼び出しを `.map_err(|e| AppError::from(e))?` パターンに統一する（マクロ）
3. `#[allow(dead_code)]` がついているすべての関数を削除または実装する（`vimgrep` + quickfix）
4. `impl` ブロックを「パブリック API → プライベートメソッド → trait 実装」の順に並べ替える（マクロ + ビジュアル選択）
5. ファイル全体をフォーマットして `cargo clippy` を通す

## 問題 2 — LSP + vimdiff + マクロの複合操作

`exercise-2/` は古いバージョンと新しいバージョンの2つの実装がある。
LSP と vimdiff を組み合わせて、v1 の実装を v2 スタイルに移行せよ。

### 条件

- `vimdiff v1/src/ v2/src/` のような形で差分を確認しながら進める
- LSP の `rename` と `code action` を積極的に使う

### 変更内容

1. vimdiff で v1 と v2 の差分をすべて確認する
2. v1 のエラーハンドリングを v2 スタイル（`thiserror` + `?` 演算子）に移行する
3. v1 の手動実装を v2 の自動 derive に置き換える（`#[derive(Debug, Clone, PartialEq, Eq)]`）
4. v1 にあって v2 にない関数を v2 スタイルで実装する
5. `cargo test` を通す（ターミナルペインで実行）

## 問題 3 — タイムアタック総合（全技術結集）

`exercise-3/` は未実装の Rust HTTP サーバースケルトン。
以下の機能を実装して `cargo build` を通すまでの時間を計れ。

### 実装する機能

1. `Config` 構造体の `new` 関数と `Display` 実装
2. `Router` の `add_route` と `dispatch` メソッド
3. `Handler` trait の基本実装（`handle_get`・`handle_post`）
4. `main` 関数でサーバーを起動する処理

### 縛り

- `hjkl` によるカーソル移動は禁止
- ファイルを開くのは netrw のみ（`:e` コマンド禁止）
