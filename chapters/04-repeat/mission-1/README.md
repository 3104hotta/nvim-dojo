# ミッション 1 — ドット（`.`）で同じ変更を複数箇所に適用

**難易度**: ★☆☆  
**目安時間**: 5〜10分

---

## 課題

`exercise.rs` には同じパターンの変更が複数箇所に必要。
1回変更したら、あとは `.` で繰り返せ。

1. `unwrap()` を `?` に変更する（4箇所）。最初の1回だけ `ciw` で変更し、残りは `n.` で
2. `pub fn` を `pub(crate) fn` に変更する（3箇所）。`fn` の前に `(crate) ` を挿入し、あとは `.` で
3. 行末の `;` を削除する（対象5行がある）。1行目で `$x` か `A<BS><Esc>` をしてから `j.` で

## 制約

- 課題1 は `:%s` での一括置換禁止。`n` で移動 → `.` で繰り返す流れを体験すること
- 各課題で `.` を使う回数を最小にすること

## ゴール

`goal.rs` と同一の状態にすること。

---

## ヒント

<details>
<summary>ヒントを見る</summary>

- `.` は「直前の変更」を繰り返す。移動は含まれない
- `ciw?<Esc>` → `/unwrap` → `n` → `.` の流れ
- 挿入・削除どちらも `.` で繰り返せる
- `n.` の組み合わせは「次のマッチへ移動して同じ変更」の最強パターン

</details>

---

## 解答例

<details>
<summary>解答を見る</summary>

```
課題1:
  /unwrap → ciw → ?<Esc>
  n.n.n.  （残り3箇所）

課題2:
  /pub fn → f  → i → (crate) <Esc>
  n.n.    （残り2箇所）

課題3:
  /対象行 → $x （行末の ; を削除）
  j.j.j.j. （残り4行）
```

</details>

---

## 別解

<details>
<summary>別解を見る（上達したら試してみよう）</summary>

- `cgn` コマンドを使うと「検索パターンにマッチした次の範囲を変更」ができ、`.` で次のマッチに自動的に進む
- `/unwrap` → `cgn` → `?<Esc>` → `.` で1回のキーで次のマッチを処理できる

</details>
