# 章末総合演習 — 3種の繰り返しを使い分けて大規模リファクタ

**難易度**: ★★★  
**目安時間**: 20〜25分

---

## シナリオ

`exercise.rs` は古い API スタイルで書かれた Rust コード（約150行）。
新しいスタイルへの移行作業を、繰り返しコマンドをフル活用して完了させよ。

## 課題

### グループ 1 — ドット繰り返し（5箇所）

`Box::new(Error::from(...))` を `Error::from(...).into()` に変更する。
最初の1回を変更したら、残りは `n.` で完了させること。

### グループ 2 — マクロ（10行）

以下のパターン変換をマクロで行う:
```
before: .map_err(|e| log::error!("{}", e))?;
after:  .map_err(|e| { tracing::error!(err = %e); e })?;
```
マクロ `qa` に記録して `9@a` で適用。

### グループ 3 — 数値プレフィクス（構造的な変更）

- `///` ドキュメントコメントが3行連続しているブロックを `5` つ見つけ、各ブロックの最初の行に `/// # Example` 行を追加する（`O` + ドット繰り返し）
- 30行目から50行目のインデントを一段増やす（`30G` → `20>>` または ビジュアル選択）

### グループ 4 — 組み合わせ

- マクロで各 `impl` ブロックの先頭に `#[allow(dead_code)]` を追加する（4箇所）
- 追加後、`u` で undo → `Ctrl+r` で redo を試して、変更履歴を確認する

## ゴール

`goal.rs` と同一の状態にすること。
