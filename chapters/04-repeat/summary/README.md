# 章末総合演習 — 3種の繰り返しを使い分けて大規模リファクタ

**難易度**: ★★★
**目安時間**: 20〜25分

---

## シナリオ

`exercise.rs`（約100行）に対して、繰り返しコマンドをフル活用して
複数箇所の同じパターンを一気に変更する。

各グループで使う繰り返し手段が異なる:
- **Group 1**: `.`（ドット繰り返し）
- **Group 2**: マクロ記録 `qa` / 適用 `@a`
- **Group 3**: 数値プレフィクス + ドット繰り返し
- **Group 4**: マクロ＋ undo/redo の確認

## 課題

### グループ 1 — ドット繰り返し（5箇所）

5つの `pub fn wrap_N(...)` 行の **末尾に ` // helper` というコメントを追加** する。

最初の1行で `A // helper<Esc>`（行末追加）を実行 → 残り4行は `j.` で繰り返し。

```
before: pub fn wrap_1(e: std::io::Error) -> Box<AppError> { Box::new(AppError::from(e)) }
after:  pub fn wrap_1(e: std::io::Error) -> Box<AppError> { Box::new(AppError::from(e)) } // helper
```

### グループ 2 — マクロ（3箇所）

`step_a` / `step_b` / `step_c` の各関数内で `log::error!` を `eprintln!` に置き換える。
マクロ `qa` に記録して `2@a` で残りに適用すること（合計3箇所）。

```
before: .map_err(|e| log::error!("{}", e))
after:  .map_err(|e| eprintln!("{}", e))
```

マクロ例:
```
qa              ← 記録開始（レジスタ a）
/log::error!<CR>← 検索でジャンプ
cgneprintln!<Esc>← 次のマッチを cgn で書き換え
q               ← 記録終了
2@a             ← 残り2箇所に適用
```

### グループ 3 — 数値プレフィクス（5箇所）

`///` ドキュメントコメントが3行連続している箇所が **5ブロック** ある。
各ブロックの **3行目（最後の `///` 行）の下** に `/// # Example` の行を追加する。

操作の流れ:
1. 最初の `///` ブロックの3行目（最後の `///` 行）にカーソルを置く
2. `o /// # Example<Esc>` で下に新しい行を追加
3. 次のブロックへジャンプ → `j.` で繰り返し（あるいは検索 `n` を併用）

**インデント練習（任意）**: `30G` でジャンプ → `20>>` で20行分のインデントを一段増やす操作を試して、`u` で元に戻す（最終結果には反映しない）。

### グループ 4 — マクロ＋ undo/redo

3つの `impl` ブロック（`impl Task`、`impl fmt::Display for AppError`、`impl From<...> for AppError`）の**直前の行**に `#[allow(dead_code)]` を追加する。

操作の流れ:
1. マクロ `qb` に「`/^impl<CR>` で検索 → `O #[allow(dead_code)]<Esc>` で上に追加」を記録
2. `2@b` で残り2箇所に適用
3. 全部追加後、`u` で1ステップずつ undo → `Ctrl+r` で redo を試す

## ゴール

`goal.rs` と同一の状態にすること。

## ヒント

<details>
<summary>ヒントを見る</summary>

- ドット繰り返し `.` は「直前の変更操作」を再実行する。`A...<Esc>` は1つの変更とみなされるので `.` で繰り返せる
- `cgn` = 次の検索マッチを変更（`/<Pattern>` 後に `cgn` で先頭マッチ→入力→Esc、`.` で次のマッチに同じ変更）
- マクロ録画中はステータスラインに `recording @a` と表示される
- `5@a` でマクロを5回連続実行
- `Ctrl+r` で undo を取り消し（redo）

</details>
