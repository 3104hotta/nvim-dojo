# 第4章 繰り返しの3種

## 概要

同じ操作を繰り返すとき、コピー&ペーストや手打ちに頼っていないか？
Vim には3種類の繰り返し機構があり、使い分けることで大量の編集が一瞬で終わる。

---

## コマンド早見表

### ① ドット（`.`）— 直前の変更を繰り返す

| コマンド | 由来 | 動作 |
|----------|------|------|
| `.` | ドット（"repeat command"）| 直前の変更操作を繰り返す |

**使いどころ**: `ciw`（単語変更）→ 次の単語へ移動 → `.` で同じ変更を適用。
**制約**: 繰り返せるのは「1回の変更操作」。移動は含まれない。
**コツ**: 「再利用しやすい変更」を意識してコマンドを設計する（例：`ciwNewWord<Esc>` を `.` で使い回す）。

### ② マクロ（`q`）— 一連の操作を記録して再生

| コマンド | 由来 | 動作 |
|----------|------|------|
| `q{a}` | レジスタ `a` への記録（`q` キー） | マクロ記録を開始 |
| `q` | 同上 | マクロ記録を終了 |
| `@{a}` | **a**t（"レジスタ `a` を実行"） | マクロを再生 |
| `@@` | 直前の `@` を繰り返す | 直前に再生したマクロを再生 |
| `{N}@{a}` | 数値プレフィクス + `@` | マクロを N 回繰り返す |
| `:{N}normal @{a}` | normal モードで実行 | N 行に対してマクロを適用 |

**使いどころ**: 同じ構造の行を複数行変換するとき。
**コツ**: マクロの末尾に `j`（次の行へ移動）を含めると `{N}@@` でN行に連続適用できる。

### ③ 数値プレフィクス（`{N}`）— N 回繰り返す

| コマンド | 由来 | 動作 |
|----------|------|------|
| `{N}{command}` | カウントプレフィクス | command を N 回実行 |
| `3w` | 3 + **w**ord | 3単語分前進 |
| `5dd` | 5 + **d**elete行 | 5行削除 |
| `10j` | 10 + 下移動 | 10行下へ移動 |
| `3>>` | 3 + インデント増 | 現在行のインデントを3レベル増加 |

**使いどころ**: 同じコマンドを連続して N 回実行したいとき。
**注意**: `{N}.` とすることで、ドットも N 回繰り返せる。

### 使い分けの基準

| 状況 | 選ぶ手段 |
|------|---------|
| 直前と「全く同じ」変更を数箇所に | `.`（ドット） |
| 移動を含む複合操作を複数回 | マクロ |
| 同じコマンドをちょうど N 回 | `{N}` プレフィクス |
