# ミッション 1 — `/` 検索と `n/N`・`*/#` でナビゲート

**難易度**: ★☆☆  
**目安時間**: 5〜10分

---

## 課題

`exercise.rs` を開き、検索コマンドだけを使ってナビゲートしながら編集せよ。

1. `handler` という単語が何箇所あるか数える（`*` で検索 → `n` で数える）
2. 最初の `handler` を `router` に変更し、残りは `n.` で変更する（`:%s` 禁止）
3. `fn handle_` で始まる関数名を順に確認する（`/fn handle_` → `n` で移動）
4. バックスラッシュ `\` を含む文字列を検索する（エスケープが必要）
5. 大文字小文字を無視して `error` を検索し（`/error\c` または `:set ignorecase`）、最初のマッチを確認する
6. 検索ハイライトを消す（`:noh`）

## 制約

- 課題2 は `:%s` 禁止。`*` で検索後に `ciw` で変更 → `n.` で繰り返すこと
- 矢印キー禁止

## ゴール

`goal.rs` と同一の状態にすること（課題2の変更が反映されていること）。

---

## ヒント

<details>
<summary>ヒントを見る</summary>

- `*` = カーソル下の単語で前方検索（単語境界つき）
- `#` = 後方検索バージョン
- `g*` = 単語境界なしの前方検索（部分一致）
- `:set hlsearch` で検索ハイライトを有効化
- `/\\` でリテラルのバックスラッシュを検索

</details>

## 解答例

<details>
<summary>解答を見る</summary>

```
1. handler の上で *
   n を押して数を数える

2. handler の上で * → ciw → router<Esc>
   n. n. n. ...

3. /fn handle_<Enter>
   n n n で順番に確認

4. /\\<Enter>

5. /error\c<Enter>

6. :noh<Enter>
```

</details>

## 別解

<details>
<summary>別解を見る（上達したら試してみよう）</summary>

- `:%s/handler/router/gn` は置換せずにマッチ数だけカウントできる（`n` フラグ）
- `/\<handler\>` で単語境界を明示して検索（`*` と同等）
- `?handler` で後方検索し `N` で前方に進む（通常の検索方向と逆になる）

</details>
