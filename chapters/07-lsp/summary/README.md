# 章末総合演習 — LSP フル活用で未知のコードを読んで改修

**難易度**: ★★★  
**目安時間**: 25〜30分

---

## シナリオ

初めて見る Rust コード `exercise/` を、LSP だけを頼りに読んで改修する。
grep や外部ツールは使わず、nvim の LSP 機能のみで完走する。

---

## 課題

### フェーズ 1 — コードリーディング（LSP ナビゲーション）

1. `main.rs` を開き、`run()` 関数を `gd` で探索する
2. `run()` が呼んでいる関数を `gd` で順にたどり、処理の全体像を把握する
3. `Config` 構造体の全参照箇所を `gr` で確認する
4. `Error` 型の定義を `gd` で見つけ、`K` でドキュメントを読む

### フェーズ 2 — 改修

5. `Config::default()` の実装が間違っているのを見つけて修正する（診断を使う）
6. `validate(&self)` メソッドを追加する（code action "Add missing impl items" を使う）
7. すべての関数にドキュメントコメント `///` を追加する（LSP hover で既存の型情報を参考にする）
8. ファイル全体をフォーマットして保存する

### フェーズ 3 — 品質チェック

9. `]d` で全診断を確認し、すべてクリアする
10. `:lua vim.diagnostic.setqflist()` で全診断を quickfix に出力し最終確認

## ゴール

- `cargo check` がエラーなしで通ること
- 診断がゼロになること
- `validate` メソッドが実装されていること
