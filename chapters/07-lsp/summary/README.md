# 章末総合演習 — LSP フル活用で未知のコードを読んで改修

**難易度**: ★★★  
**目安時間**: 25〜30分

---

## シナリオ

初めて見る Rust コード `exercise/` を、LSP だけを頼りに読んで改修する。
grep や外部ツールは使わず、nvim の LSP 機能のみで完走する。

---

## 課題

### フェーズ 1 — コードリーディング（LSP ナビゲーション）

1. `main.rs` を開き、`run()` 関数を `gd` で探索する
2. `run()` が呼んでいる関数を `gd` で順にたどり、処理の全体像を把握する
3. `Config` 構造体の全参照箇所を `gr` で確認する
4. `Error` 型の定義を `gd` で見つけ、`K` でドキュメントを読む

### フェーズ 2 — 改修

5. `Config::default()` の実装が間違っているのを見つけて修正する（診断を使う）
6. `validate(&self)` メソッドを追加する（code action "Add missing impl items" を使う）
7. すべての関数にドキュメントコメント `///` を追加する（LSP hover で既存の型情報を参考にする）
8. ファイル全体をフォーマットして保存する

### フェーズ 3 — 品質チェック

9. `]d` で全診断を確認し、すべてクリアする
10. `:lua vim.diagnostic.setqflist()` で全診断を quickfix に出力し最終確認

## ゴール

- `cargo check` がエラーなしで通ること
- 診断がゼロになること
- `validate` メソッドが実装されていること

## Rust 背景（読み飛ばし可）

総合演習なので、複数ファイルにまたがる Rust の基本構造を最低限押さえておくと LSP ナビゲーションが活きる。

- **モジュールと crate** — `mod foo;` でファイル分割、`use crate::foo::Bar` で参照。`gd` でモジュール境界を越えてジャンプできる
- **ビルダーパターン** — `Config::default().with_name("x").with_port(8080)` のようなメソッドチェーン。各 `with_*` は `self` を受け取って返すパターンが多い
- **`derive` マクロ** — `#[derive(Debug, Clone)]` などはトレイト実装を自動生成。`Debug` は `{:?}` で表示、`Clone` は `.clone()` を可能にする
- **トレイト実装の追加** — code action "Add missing impl items" でメソッドシグネチャを生成。中身は `todo!()` になっており、自分で実装する
- **エラー型の自作** — `pub enum Error { ... }` で列挙型エラーを定義し、`impl From<io::Error> for Error` で変換を提供するパターン

`Config::default()` のバグは「意図したデフォルト値と実際の値が違う」タイプ。診断には出ないので、`gd` で定義へ飛び、ホバー (`K`) や周辺コードと突き合わせて判断する必要がある。
