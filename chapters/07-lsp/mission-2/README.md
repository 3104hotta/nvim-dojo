# ミッション 2 — 診断を使ってコンパイルエラーを修正する

**難易度**: ★★☆  
**目安時間**: 10分

---

## 課題

`exercise.rs` はコンパイルエラーと警告が意図的に混入されている。
LSP の診断機能（`[d` / `]d`）を使ってすべてのエラーを修正せよ。

**混入されているエラー・警告**:
1. 型不一致: `String` を期待しているが `&str` を渡している
2. 未定義の変数を参照している
3. `Result` を処理していない（`must_use` 警告）
4. 借用チェッカーエラー: ムーブ後の値を使おうとしている
5. 未使用のインポート

## 手順

1. ファイルを開いて LSP が起動するまで少し待つ（赤い下線が表示される）
2. `]d` で最初のエラーへジャンプする
3. `:lua vim.diagnostic.open_float()` でエラーの詳細を確認する
4. エラーを修正する
5. 2〜4 を繰り返し、すべての診断をクリアする

## 制約

- `cargo check` は使っても良いが、エラー箇所への移動は `]d` / `[d` を使うこと
- コードアクション（`<leader>ca`）は次のミッションで使う。今回は手動で修正する

## ゴール

`]d` を押しても診断がなくなり、`cargo check` がエラーなしで通ること。

## Rust 背景（読み飛ばし可）

このミッションは LSP 診断（コンパイルエラー・警告）を読み解く練習。
題材で出てくるエラーの典型パターンを先に押さえておくと、`]d` で次のエラーへ飛んだとき詰まりにくい。

- **所有権とムーブ** — Rust は値を「使うと所有権が移動」するルール。`let a = b;` の後 `b` は使えなくなる。エラー: `value borrowed here after move`。直し方: `b.clone()` で複製するか、`&b` で参照を借りる
- **借用ルール** — 同じ値を「変更可能な参照 (`&mut`)」と「不変参照 (`&`)」で同時に持てない。エラー: `cannot borrow as mutable because it is also borrowed as immutable`
- **型不一致** — `&str` (文字列スライス) と `String` (所有文字列) は別の型。`String` が必要な場所に `&str` を渡すと `to_string()` 等で変換が必要
- **`#[must_use]`** — `Result` などは結果を捨てると警告。`let _ = ...;` で明示的に無視するか、`?` で扱う
- **未使用インポート** — `use std::xxx;` を書いて使わないと警告。LSP の code action で削除できる

診断メッセージは複数の解決策が示されることが多い (`clone()` か `&` か等)。
どれが「このコードの意図に合うか」は Rust 知識が必要だが、まずは LSP で **エラー箇所と提案にアクセスする操作**自体に慣れることを優先。
