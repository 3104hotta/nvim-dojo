# ミッション 2 — ネストした構造をテキストオブジェクトで編集

**難易度**: ★★☆  
**目安時間**: 10分

---

## 課題

`exercise.rs` には入れ子になった構造体・関数が含まれている。
テキストオブジェクトを使って、ネストを意識した編集を行え。

1. 外側の `impl Block` の `{}` 内にある `fn inner` 関数全体を別の場所にコピーする（`yap` または `ya{` を使う）
2. `match result {` ブロックの中身全体（`{` `}` の間）を新しいコードに置き換える（`ci{` を使う）
3. 3重ネストした `Vec<Vec<Vec<i32>>>` の型注釈を `Vec<Vec<String>>` に変更する（`ci<` を使う）
4. `println!("value: {}", some_long_variable_name)` の `some_long_variable_name` を `v` に変更する（`ci,` は使えないため別の方法で）
5. 段落（空行で区切られたコードブロック）全体をヤンクする（`yap` を使う）

## 制約

- ネストした括弧では「どの括弧が対象か」を明示的に選ぶこと
- `%` を使って括弧の対応を確認しながら進めること

## ゴール

`goal.rs` と同一の状態にすること。

---

## ヒント

<details>
<summary>ヒントを見る</summary>

- カーソルが `{` の内側にある場合、`ci{` はそのブロック内を変更する
- 複数の `{` がある場合、カーソル位置によって「どの `{` か」が変わる
- `%` で `{` に移動してからカウントを使う: `2ci{` で2つ外側の `{` を対象にできる（バージョンによる）
- `yap` は空行で囲まれた「段落」をヤンク（コードブロック1つ分をまとめてコピーするのに便利）

</details>

## 解答例

<details>
<summary>解答を見る</summary>

```
1. /fn inner → yap → ペースト先へ移動 → p
2. /match result → f{ → ci{ → 新しいコード<Esc>
3. /Vec<Vec<Vec → f< → ci< → Vec<String><Esc>
   （外側の < に合わせる）
4. /some_long → ciw → v<Esc>
5. 段落先頭で yap → ペースト先で p
```

</details>

## 別解

<details>
<summary>別解を見る（上達したら試してみよう）</summary>

- `V` で行選択 → `%` で対応する括弧まで選択範囲を拡張 → `d` で削除
- `dap` で段落ごと削除（前後の空行も含む）
- `=i{` でブロック内のインデントを自動整形

</details>
