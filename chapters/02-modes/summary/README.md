# 章末総合演習 — モードを適切に選んで最小手数で編集

**難易度**: ★★★  
**目安時間**: 15〜20分

---

## シナリオ

既存の HTTP ハンドラー実装に機能追加とリファクタを行う。
各編集で「最適なモードエントリコマンド」を選ぶことを意識せよ。

## 課題

以下の8項目をすべて完了させること:

1. `pub struct Handler` の直前に `/// HTTP リクエストを処理するハンドラー` ドキュメントコメントを追加する
2. `fn handle_get` を `pub fn handle_get` に変更する（行頭への挿入）
3. `fn handle_post` を `pub fn handle_post` に変更する（同上）
4. `fn handle_delete` を `pub fn handle_delete` に変更する（同上）
5. すべての文字列リテラル `"OK"` を `"Created"` に変更する（2箇所）
6. すべての文字列リテラル `"Not Found"` を `"Missing"` に変更する（2箇所）
7. 空実装 `{ todo!() }` の行が3行あるので、すべて `{ unimplemented!("TODO") }` に変更する
8. ファイル末尾のコメントブロック（`// --- tests ---` 以下）を矩形選択で一括コメント解除する（先頭の `// ` を削除し、`// --- tests ---` の行ごと削除）

## 制約

- 課題2〜4 は `Ctrl+v` の矩形選択 + `I` で3行まとめて変更すること（1行ずつ禁止）
- 課題5・6 は最初の1箇所を `ci"` で変更後、`n` `.` で次のマッチに繰り返し適用するか、`*` + `cgn` を使うこと
- 課題7 は `.`（ドット繰り返し）を使うこと
- 課題8 は `Ctrl+v` の矩形選択 + `x` で先頭の `// ` を一括削除すること

## ゴール

`goal.rs` と同一の状態にすること。

---

## 振り返り

- `Ctrl+v I` での一括挿入と `Ctrl+v d` での一括削除はどちらが得意か？
- `o`（下に新行）と `O`（上に新行）を使い分けられたか？
- `s`・`S`・`c`・`C` の違いを言葉で説明できるか？

| コマンド | 動作範囲 |
|----------|---------|
| `s` | 1文字 |
| `S` | 1行全体 |
| `c{motion}` | motion で指定した範囲 |
| `C` | カーソルから行末 |
