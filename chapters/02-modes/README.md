# 第2章 モードとインサート

## 概要

Vim のモデルの核心は「モード」にある。
挿入する位置・方向・範囲を意識してモードに入ることで、余計なカーソル移動がなくなる。

---

## コマンド早見表

### ノーマルモード → インサートモード

| コマンド | 由来 | カーソル位置 | 用途 |
|----------|------|------------|------|
| `i` | **i**nsert | カーソルの左 | その場で挿入 |
| `a` | **a**ppend | カーソルの右 | カーソル後に追記 |
| `I` | **I**nsert（行頭） | 行頭（非空白） | 行の先頭に挿入 |
| `A` | **A**ppend（行末） | 行末 | 行末に追記 |
| `o` | **o**pen（下） | 下に新行を作成 | 次の行に新しく書く |
| `O` | **O**pen（上） | 上に新行を作成 | 前の行に新しく書く |
| `s` | **s**ubstitute | カーソル文字を削除して挿入 | 1文字を置き換えつつ書く |
| `S` | **S**ubstitute（行） | 行全体を削除して挿入 | 行をまるごと書き直す |
| `c{motion}` | **c**hange | motion で選択した範囲 | 範囲を書き直す |
| `C` | **C**hange（末尾まで） | カーソルから行末 | 行後半を書き直す |
| `R` | **R**eplace | — | 上書きモード |

### インサートモード内の操作

| コマンド | 由来 | 動作 |
|----------|------|------|
| `Ctrl+h` | バックスペース相当 | 1文字削除 |
| `Ctrl+w` | **w**ord | 直前の単語を削除 |
| `Ctrl+u` | （行クリア） | カーソルから行頭まで削除 |
| `Ctrl+[` | ASCII で ESC と同じコード | ノーマルモードに戻る |
| `Esc` | **Esc**ape | 同上 |
| `Ctrl+o` | **o**ne normal command | 1コマンドだけノーマル操作を実行 |
| `Ctrl+r{reg}` | **r**egister | レジスタ `reg` の内容を貼り付け |

### ビジュアルモード

| コマンド | 由来 | 動作 |
|----------|------|------|
| `v` | **v**isual | 文字選択 |
| `V` | **V**isual（行） | 行選択 |
| `Ctrl+v` | **v**isual block（垂直方向）| 矩形選択 |
| `gv` | **g**o + **v**isual（再選択）| 直前の選択範囲を再選択 |
| `o` | **o**pposite（反対端） | 選択範囲の反対端にカーソルを移動 |

### モード遷移の鉄則

- **ノーマルモードが「家」**：作業が終わったら必ず戻る
- `Esc` よりも `Ctrl+[` の方が押しやすい場合が多い
- インサートモードで長く留まりすぎない（undo の粒度が粗くなる）
