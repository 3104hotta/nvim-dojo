# 章末総合演習 — Rust 4ペイン構成で総合操作

**難易度**: ★★★  
**目安時間**: 20〜25分

---

## シナリオ

実際の Rust 開発を想定した4ペイン環境を構築し、コードの編集・確認・実行を完走する。

---

## 目標レイアウト

```
┌────────────────────┬────────────────────┐
│                    │                    │
│     main.rs        │     lib.rs         │
│                    │                    │
├────────────────────┼────────────────────┤
│                    │                    │
│    Cargo.toml      │     terminal       │
│                    │                    │
└────────────────────┴────────────────────┘
```

---

## 課題

### フェーズ 1 — レイアウト構築

1. `main.rs` を開いた状態から、上記の4ペイン構成を作る
2. `Ctrl+w =` で均等化する
3. `main.rs` ペインを高さ最大化（`Ctrl+w _`）してコードを読む
4. `Ctrl+w =` で戻す

### フェーズ 2 — ファイル間の編集

5. `lib.rs` に `pub fn process(input: &str) -> Result<String, String>` を実装する
6. `main.rs` から `lib.rs` の関数を呼び出すコードを書く（`Ctrl+w h/j/k/l` でペイン移動）
7. `Cargo.toml` の `[dependencies]` セクションにコメントを追加する

### フェーズ 3 — ターミナルで確認

8. ターミナルペインで `cargo build` を実行する
9. エラーがある場合は修正する（ミッション B のサイクルを使う）
10. `cargo build` が通ったら、4ペイン全員を `:wa` で保存して完了

## ゴール

`cargo build` がエラーなしで通ること。4ペイン構成が維持されていること。

---

## 振り返りチェックリスト

| コマンド | 説明できるか |
|---------|------------|
| `Ctrl+w _` | 現在ウィンドウの高さを最大化 |
| `Ctrl+w \|` | 現在ウィンドウの幅を最大化 |
| `Ctrl+w =` | 全ウィンドウを均等化 |
| `:vertical resize {N}` | 幅をN列に指定 |
| `:resize {N}` | 高さをN行に指定 |
| `Ctrl+\ Ctrl+n` | ターミナルモードを抜ける |
