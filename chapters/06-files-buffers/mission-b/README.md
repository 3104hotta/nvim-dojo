# ミッション B — ウィンドウサイズ：エディタ ⇔ ターミナル高速シフト

**難易度**: ★★☆  
**目安時間**: 10分

---

## 概要

コードを書いて → ターミナルでビルド → 結果を確認してコードに戻る、
この往復を **最小のキーストロークで繰り返す** 練習。

---

## 課題

1. `exercise.rs` を開き、下側に `:sp | terminal` でターミナルを起動する
2. `Ctrl+w _` でエディタ（上）を最大化する
3. `Ctrl+w j` でターミナルへ移動し、`cargo check` を実行する
4. ターミナルモードを `Ctrl+\ Ctrl+n` で抜けて、`Ctrl+w =` で均等化する
5. エディタに戻り（`Ctrl+w k`）コードを修正する
6. 手順 2〜5 を3回繰り返す（往復の感覚を体に覚え込ませる）

**修正する内容**（3回分）:
- 1回目: `let x: i32 = "hello";` の型エラーを修正する
- 2回目: 未使用変数 `_unused` の警告を消すために変数を削除する
- 3回目: `fn add(a: i32, b: i32)` の戻り値型が欠けているのを追加する

## 制約

- ターミナルへの移動は `Ctrl+w j` のみ（マウス禁止）
- ターミナルモード終了は `Ctrl+\ Ctrl+n` のみ

## ゴール

3回の修正を完了し、`cargo check` がエラーなしで通ること。

---

## ヒント

<details>
<summary>ヒントを見る</summary>

- ターミナルモードでは `Ctrl+\ Ctrl+n` が「ノーマルモードに戻る」唯一の手段
- `Ctrl+w _` = 高さを最大化（ターミナルペインを小さく保ったままエディタを広げるのに便利）
- `Ctrl+w =` = 均等化（コマンド出力を読むときにターミナルを広げたい）
- インサートモードで `Ctrl+o Ctrl+w j` を使うとインサートモードのままウィンドウ移動できる

</details>

## 解答例（往復パターン）

```
初期設定:
  :sp | terminal
  i          （ターミナルモードへ）

往復サイクル:
  Ctrl+\ Ctrl+n    （ノーマルモードへ）
  Ctrl+w k         （エディタへ）
  Ctrl+w _         （エディタ最大化）
  （コードを修正）
  :w
  Ctrl+w j         （ターミナルへ）
  i                （ターミナルモードへ）
  cargo check
  Ctrl+\ Ctrl+n    （ノーマルモードへ）
  Ctrl+w =         （均等化して出力を読む）
  （繰り返し）
```
